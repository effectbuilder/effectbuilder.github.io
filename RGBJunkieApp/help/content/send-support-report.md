## When it’s time to send logs

Not everything fixes itself with a Rescan. Grab a report when:

* Gear won’t connect after sleep or replug
* Effects stutter or colors stay wrong after calibration
* An update or plugin acts weird

A zip up front saves a lot of “can you try this again?” back-and-forth.

## Create a report

1. **Settings → System → Support report** (label might vary slightly by version)
2. **Create report** or **Export**
3. RGBJunkie bundles recent **logs**, version info, and diagnostics into a zip

Save it somewhere you can attach to email or a forum post.

## Just want the log files?

**Settings → System** may have **Open logs folder**, or use [App links](app-links) to jump to `logs/` in your user data folder.

Even a few recent log files help if you don’t send the full bundle.

## What to mention when you write us

* RGBJunkie **version** — **Settings → System → About** or **Check for updates**
* Your devices — brand, USB vs WLED, IP if network
* What you already tried — [no lights](troubleshooting-no-lights), [wrong colors](troubleshooting-wrong-colors), etc.

Reports can include file paths and usernames. Don’t post them publicly if that bothers you — send through support channels on rgbjunkie.com.
