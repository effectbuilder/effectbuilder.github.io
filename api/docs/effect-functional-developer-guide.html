Functional effects (.mjs)

This page is for authors who want a single JavaScript file that drives Effect lighting without a canvas. You export a small catalog object and one color function; RGBJunkie calls that function for each LED.

For HTML + canvas effects (iframe, engine, drawing), use the HTML canvas effect guide instead.

Effect Builder, gallery, and downloads: rgbjunkie.com

What you are writing

Save one file with the extension .mjs. It must export two names:

ExportPurpose
metaPlain object: stable id, human name, optional description / publisher, and params for the sidebar.
sampleLedA function (args) => { r, g, b }. Each channel should be 0–255 (the host clamps if needed).

There is no <canvas> and no requestAnimationFrame in your module. The app passes time as args.tSec (seconds since your effect started) and tells you which LED you are with args.ledIndex, args.ledCount, and often args.stripU (0…1 along the strip).

Compared to HTML canvas effects

TopicHTML (.html)Functional (.mjs)
OutputYou draw; the host reads pixels from your canvas.You return RGB from sampleLed.
TimeYour own animation loop.args.tSec and args.frameIndex from the host.
Settings<meta property> → globals on window.meta.params → args.params[key].
Audioengine.audioargs.context.audio (same idea: 200-bin freq, level, density).
Color profilesGlobals like colorProfileStops.Add type: "colorProfile" in params, then args.context.sampleColorProfile(id, t).
Browser APIsAllowed inside the iframe.Not inside sampleLed — no window, document, or network calls there.

Minimal working example

Drop this in your user effects folder (or the app’s effects tree while developing) as e.g. MyFirstSweep.mjs. Pick a unique meta.id.

export const meta = {
  id: "my_first_sweep",
  name: "My First Sweep",
  description: "A hue that travels along the strip using host time.",
  publisher: "Your Name",
  contractVersion: 1,
  params: [
    { key: "speed", label: "Speed", type: "number", min: 1, max: 200, default: 80 },
  ],
};

function hueToRgb(p, q, t) {
  let tt = t;
  if (tt < 0) tt += 1;
  if (tt > 1) tt -= 1;
  if (tt < 1 / 6) return p + (q - p) * 6 * tt;
  if (tt < 1 / 2) return q;
  if (tt < 2 / 3) return p + (q - p) * (2 / 3 - tt) * 6;
  return p;
}

function hslToRgb(h, s, l) {
  if (s === 0) {
    const v = Math.round(l * 255);
    return { r: v, g: v, b: v };
  }
  const q = l < 0.5 ? l * (1 + s) : l + s - l * s;
  const p = 2 * l - q;
  const r = hueToRgb(p, q, h + 1 / 3);
  const g = hueToRgb(p, q, h);
  const b = hueToRgb(p, q, h - 1 / 3);
  return {
    r: Math.round(Math.max(0, Math.min(255, r * 255))),
    g: Math.round(Math.max(0, Math.min(255, g * 255))),
    b: Math.round(Math.max(0, Math.min(255, b * 255))),
  };
}

export function sampleLed(args) {
  const speed = Number(args.params.speed ?? 80);
  const n = Math.max(1, args.ledCount - 1);
  const su = args.stripU;
  const u =
    typeof su === "number" && Number.isFinite(su)
      ? Math.max(0, Math.min(1, su))
      : n <= 0
        ? 0.5
        : args.ledIndex / n;
  const phase = (args.tSec * speed * 0.012 + u) % 1;
  return hslToRgb(phase, 0.85, 0.55);
}
  • stripU is the best way to get “where on the strip” when the host provides it; otherwise divide ledIndex by ledCount - 1.
  • Shipped examples under effects/*.mjs in the RGBJunkie source tree show audio, color profiles, and more patterns.

Sidebar controls (meta.params)

Each entry needs key and type. Use label, default, min, max, and values (for lists) like you would for HTML metas.

typeControlargs.params[key]
numberSliderNumber
booleanCheckboxBoolean
colorColor pickerString #rrggbb
stringText fieldString
listDropdownString (one of values)
colorProfileSame picker as HTML effectsString profile id (e.g. Rainbow)

For colorProfile, set allowNone: true on the param if you want the None option (same idea as HTML allowNone="true").

Performance: this function may run many times per frame. Keep math tight; avoid allocating large objects per call.

Audio & color profiles

Audio — read args.context.audio when present:

  • freq: array of 200 values, roughly 0…255 per bin.
  • level: overall loudness (about -100…0).
  • density: 0…1 “how much spectrum energy” for simple reactions.

Color profiles — add a param { key: "colorProfile", type: "colorProfile", default: "Rainbow" }. At runtime, call:

const fn = args.context && args.context.sampleColorProfile;
if (typeof fn === "function") {
  const id = String(args.params.colorProfile ?? "Rainbow");
  const rgb = fn(id, t01WrappedTo0Through1);
}

Built-in effects tint their math through the profile so colors stay on the user’s palette (often by scaling profile RGB by the luminance of the color they would have drawn without a profile).

Rules & tips

  • Pure sampleLed: no DOM, no window, no fetches, no timers — only math from the arguments.
  • One file: put helpers in the same module; avoid importing unknown URLs.
  • Multi-tab workspaces use the same saved params model as HTML effects; each tab can remember its own effect.

Install your file

Use the same user effects folder (or product-specific path) as for .html effects. The catalog picks up .mjs automatically. Keep the file basename unique next to any .html in the same folder.
