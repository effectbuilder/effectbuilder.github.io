Component catalog (JSON)

This page uses the same in-browser Firebase connection as the Component Builder (no Node, no npm, no Cloud Functions). It loads public components from the cloud, then builds JSON with stable id / shareId / catalogId, normalized LedCount, ProductName, DisplayName, Brand, Type, ImageUrl (merging common field name variants from Firestore), plus BuilderUrl and SignalrgbJsonUrl on https://rgbjunkie.com. Choose the root shape in the dropdown; nested mode uses { "id", "component": { … } } with ids only on the wrapper.

Click “Build catalog” to load from Firestore.

Optional URL switches: ?format=components|array|items|data|nested; ?autostart=1; ?download=1; ?max=3000; ?includeDataImages=1.
