Functional Effects (.mjs)

Welcome! This guide is for creators who want to build lighting effects using a single, lightweight JavaScript file. There is no need to draw on a canvas or manage animation loops. You simply provide a small settings object and one color function, and RGBJunkie takes care of the rest—calling your function to light up every single LED!

If you prefer working with HTML and canvas drawing (using iframes and the engine API), check out our friendly HTML canvas effect guide instead.

Effect Builder, gallery, and downloads: rgbjunkie.com

The Short Version

You export two things: meta (to give your effect a name and some sidebar controls) and sampleLed (a function that decides the RGB color for one LED). RGBJunkie does the heavy lifting, calling that function thousands of times per second to bring your lights to life.

How it works

Getting started is easy. Just save a single file with an .mjs extension. Inside, your code needs to export these two items:

ExportWhat it does
metaA simple object containing your effect's stable id, a human-readable name, an optional description, your name as the publisher, and any params (sliders/toggles) you want in the sidebar.
sampleLedYour core logic! It's a function formatted as (args) => { r, g, b }. It returns the color for a specific LED, with each color channel ranging from 0 to 255.

Because there is no <canvas> and no requestAnimationFrame loop to manage, the app hands you everything you need directly. You'll get the current time via args.tSec, and you'll know exactly which LED you are coloring via args.ledIndex, args.ledCount, and args.stripU (which tells you how far along the strip the LED is, from 0 to 1).

Compared to HTML canvas effects

Not sure which method to use? Here is a quick breakdown of how Functional effects differ from HTML ones:

TopicHTML (.html)Functional (.mjs)
OutputYou draw shapes; the host reads pixels from your canvas.You calculate and return RGB values directly from sampleLed.
TimeYou manage your own animation loop.The host feeds you args.tSec and args.frameIndex.
Settings<meta property> tags become global variables on window.You define meta.params, which show up in args.params[key].
AudioRead from engine.audio.Read from args.context.audio. It has the same shape: freq, level, density, etc.
Color profilesGlobal arrays like colorProfileStops.Add type: "colorProfile" to your params, then use args.context.sampleColorProfile(id, t).
Browser APIsFully allowed inside the iframe.Not allowed inside sampleLed. You cannot access window, document, or make network calls. Keep it pure math!

Your first effect

Want to see it in action? Drop this code into your user effects folder (e.g., MyFirstSweep.mjs). Just make sure to pick a unique meta.id!

export const meta = {
  id: "my_first_sweep",
  name: "My First Sweep",
  description: "A friendly, traveling hue that moves along the strip.",
  publisher: "Your Name",
  contractVersion: 1,
  params: [
    { key: "speed", label: "Speed", type: "number", min: 1, max: 200, default: 80 },
  ],
};

// A handy helper to convert Hue into RGB values
function hueToRgb(p, q, t) {
  let tt = t;
  if (tt < 0) tt += 1;
  if (tt > 1) tt -= 1;
  if (tt < 1 / 6) return p + (q - p) * 6 * tt;
  if (tt < 1 / 2) return q;
  if (tt < 2 / 3) return p + (q - p) * (2 / 3 - tt) * 6;
  return p;
}

// Converts standard HSL (Hue, Saturation, Lightness) to our required RGB format
function hslToRgb(h, s, l) {
  if (s === 0) {
    const v = Math.round(l * 255);
    return { r: v, g: v, b: v };
  }
  const q = l < 0.5 ? l * (1 + s) : l + s - l * s;
  const p = 2 * l - q;
  const r = hueToRgb(p, q, h + 1 / 3);
  const g = hueToRgb(p, q, h);
  const b = hueToRgb(p, q, h - 1 / 3);
  return {
    r: Math.round(Math.max(0, Math.min(255, r * 255))),
    g: Math.round(Math.max(0, Math.min(255, g * 255))),
    b: Math.round(Math.max(0, Math.min(255, b * 255))),
  };
}

// The star of the show: this runs for every LED!
export function sampleLed(args) {
  const speed = Number(args.params.speed ?? 80);
  const n = Math.max(1, args.ledCount - 1);
  const su = args.stripU;
  
  // Find out where we are on the strip (0.0 to 1.0)
  const u =
    typeof su === "number" && Number.isFinite(su)
      ? Math.max(0, Math.min(1, su))
      : n <= 0
        ? 0.5
        : args.ledIndex / n;
        
  // Calculate the color phase based on time, speed, and position
  const phase = (args.tSec * speed * 0.012 + u) % 1;
  
  return hslToRgb(phase, 0.85, 0.55);
}
  • stripU is the easiest way to figure out where an LED sits on the strip. If it's not provided, you can always calculate it by dividing ledIndex by ledCount - 1.
  • Feel free to explore the built-in examples in the RGBJunkie effects/*.mjs folder to see how to integrate audio and color profiles!

Setting up meta

FieldRequiredNotes
idYesYour effect's unique ID. Use lowercase letters, numbers, and underscores (e.g., my_cool_effect).
nameYesThe nice, readable title that users will click on in the effect list.
paramsNoAn array of sliders, checkboxes, and colors to show in the sidebar (details below).
descriptionNoA helpful blurb shown in tooltips.
publisherNoLet people know you made it!
contractVersionNoJust leave this at 1.

Adding sidebar controls (meta.params)

Adding controls is a breeze. Each control needs a key (how you reference it in your code) and a type. You can also define a label, default, min, max, and values.

typeWhat the user seesWhat you get in args.params[key]
numberA sliderA Number
booleanA checkboxA Boolean (true/false)
colorA color pickerA String like #rrggbb
stringA text fieldA String
listA dropdown menuA String (whichever option they picked)
colorProfileA color profile pickerA String ID (e.g., Rainbow)

Pro tip: For colorProfile, you can add allowNone: true to your parameter if you want users to have the option to turn the profile off.

The sampleLed(args) function

Here is all the data RGBJunkie hands you inside the args object:

PropertyMeaning
tSecHow many seconds have passed since the effect started running. Great for animations!
frameIndexThe current frame number.
ledIndex / ledCountExactly which LED you are currently coloring, out of the total.
layout{ x, y } positions on the visual workspace, if you need 2D spatial awareness.
stripUA handy 0…1 decimal showing the LED's position along the strip.
paramsThe live values of all the sidebar controls you set up in your meta.
contextBonus features! Audio data and color profiles live here.

Performance tip: Remember that sampleLed runs a lot—thousands of times a second! Keep your math clean and try to avoid creating large arrays or objects inside the function.

Audio & color profiles

Audio (mono mix)

Want your lights to react to music? Inside sampleLed, just peek at args.context.audio. It contains these core properties:

freq
An array of 200 spectrum values (0–255), representing frequencies from deep bass to high treble.
level
The overall volume/loudness (ranging from roughly −100 to 0).
density
A simple 0 to 1 value representing how much energetic sound is currently playing.

Stereo (left & right)

If the user has stereo audio configured, stereo will be true, and you'll get access to left and right channels separately! (If they are on mono, just stick to using freq so everything looks even).

freqL, freqR
The split 200-bin arrays for the left and right channels.
levelL, levelR, densityL, densityR
Volume and energy, split by channel.
pan
A simple panning value from −1 (hard left) to +1 (hard right).

Example: Lighting the left side of the strip with the left audio channel:

const a = args.context && args.context.audio;
if (!a || !Array.isArray(a.freq) || a.freq.length < 2) return { r: 0, g: 0, b: 0 };

const stereo =
  !!a.stereo &&
  Array.isArray(a.freqL) && Array.isArray(a.freqR) &&
  a.freqL.length === a.freq.length &&
  a.freqR.length === a.freq.length;

const u = typeof args.stripU === "number" ? args.stripU : 0.5;

// If we are on the first half of the strip (u < 0.5), use the left channel!
const freq = u < 0.5
  ? (stereo ? a.freqL : a.freq)
  : (stereo ? a.freqR : a.freq);

Color profiles

To let users apply their favorite color themes, add a parameter to your meta like this: { key: "colorProfile", type: "colorProfile", default: "Rainbow" }. Then, inside your code:

const fn = args.context && args.context.sampleColorProfile;
if (typeof fn === "function") {
  const id = String(args.params.colorProfile ?? "Rainbow");
  // Pass your desired position (0 to 1) to get a beautiful RGB value back!
  const rgb = fn(id, position0to1); 
}

Rules & tips for success

  • Keep it pure: sampleLed shouldn't try to manipulate the DOM, use window, fetch internet data, or use timers. Only use the math from your args.
  • Everything in its place: Put all your helper functions inside the same `.mjs` file. Don't try to import unknown external URLs.
  • Play well with tabs: Users can open multiple workspace tabs, and each tab will remember its own specific settings for your effect!

Testing your file

Ready to see your creation? Just save your `.mjs` file into your user effects folder (the same place you'd put `.html` effects). RGBJunkie's catalog will detect it automatically. Just make sure the file name is unique!
