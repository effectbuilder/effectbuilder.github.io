Welcome to the RGBJunkie Effect Developer Guide!

Have you ever wanted to design your own custom lighting patterns? If you want to build HTML canvas effects that run when users choose the Effect lighting mode in the RGBJunkie app, you're in exactly the right place. Even if you are a complete beginner still getting the hang of JavaScript, you can absolutely do this. If that sounds like you, start with our friendly Simple motion & shapes section, and don't forget to play with the worked examples below!

Once you've got the hang of things, the rest of this page acts as your handy technical reference for things like metadata, sidebar controls, and the engine API. You don't even need access to the RGBJunkie source code to create amazing things.

Note: Functional (.mjs) effects use a single JavaScript module instead of a canvas. If you're looking for that, head over to the functional effects guide. This page is all about the magic of the HTML canvas!

Making simple effects (Start here!)

RGBJunkie effects are just regular web pages with a <canvas> tag. Think of it like a flipbook: you draw one frame at a time, and the app translates your drawing directly onto your LEDs. You do not need to be a math genius — some of the most beautiful effects are just a few basic shapes moving in a repetitive pattern.

Your first animation loop

Modern web browsers have a wonderful feature called requestAnimationFrame. It lets you tell the computer, "Hey, run this function right before you draw the next frame on the screen" (which happens about 60 times a second). Inside that function, you just clear the canvas, draw your shapes, and ask for the next frame. It's that easy!

  • const ctx = canvas.getContext("2d") — Think of this as grabbing your digital "pencil" to draw rectangles, circles, and colors.
  • canvas.width / canvas.height — The app automatically tells you how big your canvas is. Always use these to figure out where to draw so your effect scales beautifully! (§2.1, §2.2).

Drawing basic shapes

Here are the common building blocks you'll use over and over:

  • A solid rectangle — ctx.fillStyle = "#3399ff"; ctx.fillRect(x, y, width, height);
  • A smooth circle — ctx.beginPath(); ctx.arc(cx, cy, radius, 0, Math.PI * 2); ctx.fill();
  • A glowing line — ctx.beginPath(); ctx.moveTo(x1, y1); ctx.lineTo(x2, y2); ctx.stroke();
  • Vibrant colors — Build a color string like "hsl(" + hue + ", 80%, 55%)" (hue goes from 0 to 360) and assign it to ctx.fillStyle.

Why Math.sin and Math.cos are your best friends

Don't let the trigonometry scare you! Math.sin and Math.cos are just tools that always hand you a number between −1 and 1. If you feed them a number that slowly gets bigger (like a timer), they will gently rock back and forth like a pendulum. This is perfect for making things move smoothly or pulsing colors without jarring jumps.

  • Orbiting around: Pick a center (cx, cy), a radius R, and a time t. Then x = cx + Math.cos(t) * R and y = cy + Math.sin(t) * R. Increase t, and watch your dot orbit perfectly!
  • Sliding left and right: x = cx + Math.sin(t) * distance (no cos needed!). Do the same with y to slide up and down.
  • Breathing lights: You can turn that −1 to 1 into a gentle 0 to 1 pulse like this: const u = Math.sin(t) * 0.5 + 0.5. Now use u to slowly fade colors in and out!

Here is a minimal, working animation you can drop right into your script:

const canvas = document.getElementById("fx");
const ctx = canvas.getContext("2d");

function draw() {
const t = performance.now() / 1000;
const w = canvas.width, h = canvas.height;
const cx = w * 0.5, cy = h * 0.5;
const R = Math.min(w, h) * 0.28;
const x = cx + Math.cos(t) * R;
const y = cy + Math.sin(t) * R;

ctx.fillStyle = "#0a0a12";
ctx.fillRect(0, 0, w, h);
ctx.beginPath();
ctx.arc(x, y, 14, 0, Math.PI * 2);
ctx.fillStyle = "hsl(" + ((Math.sin(t * 0.6) * 0.5 + 0.5) * 360) + ", 85%, 58%)";
ctx.fill();

requestAnimationFrame(draw);
}
draw();

1. What exactly is an effect?

  • It's just a single HTML document. When a user picks your effect, the app safely loads your page into an invisible frame.
  • To help you out, the app hands you a few special tools automatically: window.engine, window.points, and any settings you declare with <meta property> tags (§3, §4).
  • You draw your frame to the ``, and RGBJunkie's engine quietly handles turning those pixels into LED commands. You just focus on the art!

2. Your first template

Here is the bare minimum code you need to get started. Notice how the CSS makes the canvas fill the whole screen. RGBJunkie will handle figuring out the actual pixel resolution for you.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8" />
  <title>My Awesome Effect</title>
  <meta name="developer" content="Your Name Here" />
  <meta name="description" content="A short, catchy description for the gallery." />
  <meta name="tags" content="ambient, gradient, fun" />

  <meta property="speed" label="Speed" type="number" min="1" max="100" default="40" />

  <style>
    html, body { margin: 0; height: 100%; overflow: hidden; background: #000; }
    canvas { display: block; width: 100%; height: 100%; }
  </style>
</head>
<body>
  <!-- Leave the width and height blank; the app manages this for you! -->
  <canvas id="fx"></canvas>
  <script>
    const canvas = document.getElementById("fx");
    const ctx = canvas.getContext("2d");
    
    function draw() {
      const w = canvas.width, h = canvas.height;
      // Draw your magic here!
    }
    
    function frame() { 
      requestAnimationFrame(frame); 
      draw(); 
    }
    
    // Start the loop!
    requestAnimationFrame(frame);
  </script>
</body>
</html>

2.1 Canvas and workspace container

Your effect needs to flexibly fill whatever space the app gives it in the workspace preview. Using CSS width: 100% and height: 100% ensures your canvas is always the perfect size.

When the app changes the resolution, it automatically fires a resize event on your window, scaling your bitmap so things always look sharp.

SourceWhat it means for you
canvas.width, canvas.heightThe actual pixels available right now — always use these for drawing.
engine.canvas.width, engine.canvas.heightThe logical Engine Resolution the user set in their preferences.

2.2 Making shapes scale nicely

Because users can change their Engine Resolution, it's a good habit to tie the sizes of your shapes to the canvas size, rather than using hard-coded numbers (like `radius = 40`). That way, if a user makes their layout huge or tiny, your effect still looks perfect!

Option 1: Fractions of width/height (Super reliable!)

const w = canvas.width, h = canvas.height;
const cx = w * 0.5, cy = h * 0.5;
// The radius will always be 35% of the shortest side!
const radius = Math.min(w, h) * 0.35;
ctx.arc(cx, cy, radius, 0, Math.PI * 2);

Option 2: Using the app's uniform scale helper

const w = Math.max(1, canvas.width | 0);
const h = Math.max(1, canvas.height | 0);
const layout = typeof rgbjSetupCanvas === "function"
  ? rgbjSetupCanvas(canvas)
  : { width: w, height: h, scale: Math.min(w / 320, h / 200) };
  
const dotStep = 8 * layout.scale;
ctx.lineWidth = 2 * layout.scale;

3. Meta tags

RGBJunkie reads ordinary HTML <meta> tags from your effect's <head>. You do not need a separate config file — the tags are the configuration. The app uses them to label your effect in the library, build sidebar controls, and inject values into your script as global variables.

3.1 Overview — two kinds of tags

There are two families. Both live in <head>, but they do different jobs:

KindHow you write itWhat RGBJunkie does with it
Catalog tags <meta name="…" content="…"> Shown in the effect browser and gallery — title, author, description, search tags. They are not exposed as JavaScript variables.
Setting tags <meta property="…" … /> Builds a sidebar control and injects a global variable with the same name as property (e.g. speed, colorProfile). See §4.

Your effect's display name comes from the document <title> element (not a meta tag). When the user changes a setting, the app updates the matching global and your draw() loop can read it on the next frame.

Minimal example — catalog info plus one slider:

<head>
  <meta charset="utf-8" />
  <title>My Breathing Glow</title>
  <meta name="developer" content="Your Name" />
  <meta name="description" content="A soft pulsing ambient look." />
  <meta name="tags" content="ambient, calm, gradient" />

  <meta property="speed" label="Speed" type="number" min="1" max="100" default="40" group="Motion" />
</head>

In your script, use the setting like any other variable: const rate = speed / 50;

3.2 Catalog tags (discovery)

These tags help users find and recognize your effect in the RGBJunkie library and on rgbjunkie.com. They do not create sidebar controls.

TagExamplePurpose
<title><title>Ocean Pulse</title>Effect name shown in the app and gallery.
name="developer"content="Ada Lovelace"Author credit.
name="description"content="Slow waves of blue and teal."Short blurb on cards and detail views.
name="tags"content="ambient, ocean, calm"Comma-separated keywords for search and filters.

3.3 Sidebar layout (group, groupOrder, tooltip)

Setting tags can be organized in the effects sidebar. Use these optional attributes on <meta property> lines:

AttributeExampleWhat it does
group="…"group="Motion"Puts the control under a category tab (Motion, Colors, Audio, etc.). Settings with the same group appear together.
name="groupOrder"content="Display, Look, Processing"Catalog meta (not a property) that sets the order of group tabs. Useful when some groups are injected by the host (see screen capture in §6.4).
tooltip="…"tooltip="Higher = faster orbit."Help text shown when the user hovers the control label.

Some APIs add their own sidebar controls automatically — you do not declare meta tags for them. For example, calling engine.requestNativeScreenBitmap() adds Display controls (monitor, capture rate, size). Put your own settings in groups like Look or Processing and use groupOrder if you need a specific tab order.

4. User settings (property tags)

Each <meta property="…"> line declares one user-facing control. RGBJunkie renders it in the sidebar and keeps a global variable in sync with the user's choice. Use the attribute name property (not name) for settings — that is how the app tells catalog tags apart from controls. 

<meta property="speed" label="Speed" type="number" min="1" max="100" step="1" default="40" group="Motion" tooltip="Animation speed." />

Read settings directly in JavaScript — no window. prefix required in most effects: if (mirrorBars) { … }, ctx.fillStyle = customColor;. Booleans arrive as true / false; numbers as JavaScript numbers; colors and dropdowns as strings.

6. The `engine` API (Your superpowers)

The engine object is where the real fun begins! It lets you peek at the computer's audio playing, screen pixels, and even PC temperatures without you having to write any complicated capture code.

6.1 Audio — Making lights dance to music

Every single frame, you can read engine.audio. The app updates this roughly 30 times a second just by listening to whatever is playing on the computer. It's incredibly fun to build with!

freq
An array of 200 numbers going from bass to treble. Each number is between 0 and 255. This is all you need to build classic visualizer bars!
level
Overall loudness. Great for making a central logo pulse to the beat.
density
A quick 0–1 value telling you how much energy is in the music right now.

6.2 Stereo Audio (Left and Right)

If the computer supports stereo capture, engine.audio.stereo will be true, unlocking separate arrays for the left and right speakers. Don't worry, if stereo isn't available, you can safely fall back to the mono freq array.

freqL, freqR
The same 200 bins as above, but separated by ear!
pan
Tells you if the sound is leaning left (-1) or right (+1).

6.3 Canvas helpers

engine.canvas.width, engine.canvas.height
The logical resolution from the user's settings. Always use your element's actual width/height for drawing, though!
rgbjSetupCanvas(canvas)
A super handy function that returns an easy scale factor to keep your shapes proportional.

6.4 Screen capture — Ambient lighting

Want your lights to match the movie or game the user is playing? Just ask the engine for a screenshot! When you use these commands, the app automatically adds "Capture Monitor" and "FPS" controls to the sidebar so the user is in complete control.

engine.requestNativeScreenBitmap() (Highly Recommended)
Asks for an image bitmap you can immediately draw to your canvas using drawImage. It's fast and easy!
engine.requestNativeScreenFrame()
Gives you raw pixel data if you want to write your own code to analyze dominant colors on the screen.

6.5 PC sensors — CPU and Temperatures

On Windows, you can react to how hard the computer is working! Set a metadata type to type="sensor", and you can pull live thermal and fan data straight from LibreHardwareMonitor.

engine.sensors
Provides list(), get(id), and normalize(id) so you can seamlessly link LED colors to system temps.

7. Rendering Best Practices

  • Draw big and bold to the <canvas>! The app handles shrinking it down to match the LEDs.
  • Let CSS handle stretching the canvas to fit the iframe, and listen for resize events.
  • Always use requestAnimationFrame so your effect runs smoothly and saves resources.

8. Keyboard Interactive Effects

  • For privacy, RGBJunkie only ever sends physical key locations (KeyboardEvent.code), never the actual letters being typed.
  • You can map these codes to coordinates on your canvas and trigger ripples, flashes, etc. when keys are pressed!

9. Troubleshooting

Stuck on a bug? Any time you use console.log(), console.warn(), or console.error(), it goes straight to the RGBJunkie host effect log. It's a great way to debug without needing browser dev tools open.

12.1 Color profile gradient

How it works: By adding the type="colorProfile" setting, the app magically gives us an array called colorProfileStops. This array contains all the hex colors we need to draw a beautiful, seamless gradient. In this example, we draw the gradient twice as wide as the screen and slowly slide it over so it looks like it loops forever!

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8" />
  <title>Color profile gradient</title>
  <meta name="description" content="Animated gradient driven by the user's selected color profile." />

  <!-- The app turns this one line into a full color-profile picker for the user! -->
  <meta property="colorProfile" label="Color profile" type="colorProfile" default="Rainbow" />

  <style>
    html, body { margin: 0; height: 100%; background: #000; }
    canvas { display: block; width: 100%; height: 100%; }
  </style>
</head>
<body>
  <canvas id="fx"></canvas>
  <script>
    const canvas = document.getElementById("fx");
    const ctx = canvas.getContext("2d");

    function draw() {
      const stops = colorProfileStops;
      const w = canvas.width, h = canvas.height;
      const slide = (performance.now() / 5000) % 1;

      // Make the gradient huge so it slides seamlessly
      const grad = ctx.createLinearGradient(-slide * w, 0, (2 - slide) * w, 0);
      for (let i = 0; i < stops.length; i++) {
        const p = (i / (stops.length - 1)) * 0.5;
        grad.addColorStop(p, stops[i]);
        grad.addColorStop(p + 0.5, stops[i]);
      }

      ctx.fillStyle = grad;
      ctx.fillRect(0, 0, w, h);
      requestAnimationFrame(draw);
    }
    draw();
  </script>
</body>
</html>

12.2 Audio spectrum bars

How it works: The engine provides 200 raw data points representing audio frequencies. That's a lot of data! To make it look like a classic equalizer, we group those 200 data points down into just 64 visual bars. Notice how we paint a fresh black rectangle fillRect over everything every frame. This stops the colors from smearing together like a paint trail!

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8" />
  <title>Audio bars</title>
  <meta name="description" content="Vertical bars driven by the audio spectrum." />

  <style>
    html, body { margin: 0; height: 100%; background: #000; }
    canvas { display: block; width: 100%; height: 100%; }
  </style>
</head>
<body>
  <!-- A locked resolution helps the bars look crisp -->
  <canvas id="fx" width="640" height="320"></canvas>
  <script>
    const canvas = document.getElementById("fx");
    const ctx = canvas.getContext("2d");
    const BARS = 64;

    function draw() {
      // Grab the audio data provided by the app
      const freq = engine.audio.freq;
      const w = canvas.width, h = canvas.height;
      const barW = w / BARS;

      // Clean the slate with a solid black background
      ctx.fillStyle = "#000";
      ctx.fillRect(0, 0, w, h);
      
      // Draw our equalizer!
      for (let i = 0; i < BARS; i++) {
        const v = freq[Math.floor(i * freq.length / BARS)] / 255;
        ctx.fillStyle = `hsl(${(i / BARS) * 360}, 80%, 55%)`;
        ctx.fillRect(i * barW, h - v * h, barW, v * h);
      }
      requestAnimationFrame(draw);
    }
    draw();
  </script>
</body>
</html>

12.3 Screen sample

How it works: This is almost surprisingly easy. By calling engine.requestNativeScreenBitmap(), the app takes care of capturing the monitor, handling user settings, and managing performance. All your script has to do is wait for the picture to arrive (await), draw it to the canvas, and ask for the next one.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8" />
  <title>Screen sample</title>
  <meta name="description" content="Draws a downscaled copy of the user's desktop into the canvas." />

  <style>
    html, body { margin: 0; height: 100%; background: #000; }
    canvas { display: block; width: 100%; height: 100%; }
  </style>
</head>
<body>
  <canvas id="fx"></canvas>
  <script>
    const canvas = document.getElementById("fx");
    const ctx = canvas.getContext("2d");

    async function loop() {
      // The app handles the heavy lifting, we just ask for the frame!
      const bmp = await engine.requestNativeScreenBitmap();
      ctx.drawImage(bmp, 0, 0, canvas.width, canvas.height);
      bmp.close(); // Clean up memory
      requestAnimationFrame(loop);
    }
    loop();
  </script>
</body>
</html>
